## 1. 引言

### 1.1 项目背景
小禾帮教务管理系统（以下简称“系统”）是湖北云铺网络科技有限公司旗下产品，面向艺术培训、舞蹈、球类、兴趣班、补习班等教育培训机构，提供学员管理、招生营销、教务排课、考勤签到、家校互动、财务管理等一体化解决方案。

### 1.2 测试目标
- 验证系统功能是否符合需求规格，覆盖所有核心业务模块。
- 确保系统在预期负载下性能稳定、响应及时。
- 保证数据安全性、权限控制正确性及系统可靠性。
- 发现并跟踪缺陷，确保发布质量达到上线标准。
- 评估用户体验，保证关键流程（排课、点名、家校互动）操作顺畅。
- **建立基于 pytest + Selenium + Allure 的自动化回归测试体系**，提升回归效率。

### 1.3 测试范围
**包含模块：**
- 学员管理（增删改查、剩余课时、积分、跟进记录）
- 招生管理（线索管理、渠道标签、试听申请）
- 排课管理（可视化排课、图形课表、调课补课）
- 教务管理（课程/班级/教师/教材管理，计费方式）
- 考勤签到（刷卡、刷脸、手机点名、请假处理）
- 家校互动（家长端查看课表、请假、评价、积分兑换）
- 财务管理（缴费、退费、续费提醒、收支报表）
- 系统管理（机构/角色/权限配置、日志）

**不包含范围：**
- 服务器硬件选型及机房环境
- 第三方支付接口的合规性审计（仅测试接口连通性及数据交换）
- 非功能性安全渗透测试（如需专项安全测试，另行安排）

## 2. 测试策略

### 2.1 测试层级与类型

| 测试类型 | 覆盖范围 | 执行方式 | 通过准则 |
|----------|----------|----------|----------|
| **单元测试** | 开发自测关键业务逻辑（如课时计算、排课冲突检测） | 开发 + pytest | 覆盖率 ≥ 80%，所有用例通过 |
| **集成测试** | 模块间接口（如考勤→扣课时→家长端通知） | pytest + requests 库 | 接口返回正确，数据一致性无异常 |
| **功能测试** | 全量需求功能点（含正向/逆向/边界） | 手工 + **pytest + Selenium**（UI自动化） | 用例通过率 100%，严重及以上级别缺陷为 0 |
| **UI 测试** | 页面布局、文案、交互响应、移动端适配 | 手工 + 响应式测试工具 | 无明显错位、错别字、失效链接 |
| **兼容性测试** | 浏览器（Chrome/Firefox/Edge/360极速）<br>移动端（iOS/Android 微信内置浏览器） | 云测平台 + 真机 | 核心流程在各环境下可正常操作 |
| **性能测试** | 并发排课、高峰期考勤签到、家长端查看课表 | JMeter/LoadRunner | 200 并发下，关键接口响应 ≤ 2s，CPU/内存 ≤ 75% |
| **安全测试** | 登录认证、越权访问、SQL 注入、XSS | 手工 + 漏扫工具（AppScan） | 无高危/严重漏洞，敏感数据不泄露 |
| **可靠性测试** | 长时间运行（7×24h）、异常断电/断网恢复 | 自动化脚本 + 混沌工程 | 系统自动恢复，数据不丢失 |
| **验收测试** | 基于真实业务场景的端到端流程（机构管理员视角） | 业务方参与 + 测试团队 | 所有验收场景通过，用户满意度 ≥ 90% |

### 2.2 自动化测试技术栈（核心）

| 组件 | 选型 | 用途 |
|------|------|------|
| 编程语言 | Python 3.10+ | 测试脚本开发 |
| 测试框架 | pytest | 用例组织、断言、夹具、参数化、插件体系 |
| UI 自动化 | Selenium WebDriver | 浏览器操作、页面交互 |
| 元素定位 | 显式等待 + Page Object 模式 | 提高脚本稳定性和可维护性 |
| 报告生成 | Allure | 生成美观、信息丰富的测试报告（含步骤、截图、日志） |
| 断言库 | pytest 原生断言 + hamcrest（可选） | 丰富断言语义 |
| 数据驱动 | pytest 参数化 + YAML/Excel/CSV | 分离测试数据与脚本 |
| 日志与截图 | logging 模块 + selenium 截图 | 辅助定位失败原因 |
| CI 集成 | Jenkins / GitLab CI | 定时触发、邮件通知、Allure 报告发布 |
| 版本控制 | Git | 脚本及配置管理 |

### 2.3 测试环境

| 环境 | 用途 | 配置 |
|------|------|------|
| 开发环境 | 开发自测、单元测试 | 开发人员本地 |
| 测试环境 | 功能测试、集成测试、大部分非功能测试 | 腾讯云 4C8G，MySQL 5.7，Redis，Nginx |
| 自动化执行环境 | 定时/触发执行 pytest + Selenium 脚本 | 独立虚拟机/容器（无头浏览器） |
| 预生产环境 | 验收测试、性能摸底、演练 | 与生产 1:1 配置（8C16G + 读写分离） |
| 生产环境 | 正式上线后监控 | 客户实际部署环境 |

**测试数据：**
- 构造 5 个机构（含单校区/多校区）
- 学员数据：不少于 1000 条，含不同状态（试听/在读/停课/毕业）
- 课程数据：含一对一、一对多、集体班，按期/按课时计费
- 考勤记录：含正常、迟到、请假、缺勤

### 2.6 测试优先级

- **P0（最高）**：登录/权限、排课核心逻辑、考勤扣费、财务计费、家校通知发送 → 阻塞流程，必须修复才能继续测试
- **P1**：学员管理、招生线索转化、报表导出、批量操作 → 主要业务功能，影响用户使用体验
- **P2**：UI 样式、提示文案、次要查询排序 → 建议修复，但不影响核心流程
- **P3**：易用性建议、优化类需求 → 可排入后续迭代

## 5. 风险管理

| 风险类型 | 描述 | 可能性 | 影响 | 应对措施 |
|----------|------|--------|------|----------|
| 需求变更 | 排课规则或计费方式频繁调整 | 中 | 高 | 约定变更流程，冻结需求前进行影响评估；自动化脚本采用 PageObject，降低维护成本 |
| 开发延期 | 核心模块（如智能排课算法）开发超期 | 中 | 高 | 提前介入测试左移，对已交付模块优先测试；预留缓冲时间 |
| 测试环境不稳定 | 数据库或服务频繁重启 | 低 | 中 | 建立环境健康检查脚本，每天环境备份，出现故障快速回滚 |
| 数据准备不足 | 缺少边界或异常数据（如剩余课时负数场景） | 低 | 中 | 采用数据工厂模式，自动生成全覆盖测试数据 |
| 并发性能瓶颈 | 多家机构同时排课或点名时响应慢 | 中 | 高 | 提前进行单模块压测，发现瓶颈后协同开发优化 SQL 及缓存策略 |
| 家长端兼容性问题 | 不同手机型号/微信版本显示异常 | 中 | 中 | 使用云真机覆盖 Top 30 机型，建立快速反馈通道 |
| **Selenium 脚本稳定性** | 元素未加载、网络延迟、动态 ID | 高 | 中 | **使用显式等待（WebDriverWait）、封装重试机制、采用相对定位及 CSS 选择器** |
| **Allure 报告过大** | 长期运行后历史数据占磁盘 | 低 | 低 | **配置 Jenkins 保留最近 30 次报告，定期清理** |

## 6. 缺陷管理

### 6.1 缺陷严重程度定义

- **致命**：系统崩溃、数据丢失、支付错误、主要流程不可用（如无法排课）
- **严重**：功能未实现或与需求严重不符、数据计算错误（如课时扣错）
- **一般**：边界条件错误、界面显示异常、提示信息不准确
- **轻微**：拼写错误、对齐问题、优化建议

### 6.2 缺陷生命周期

新建 → 确认 → 分配 → 修复中 → 已修复 → 回归验证 → 关闭  
（对于“暂不修复”或“重复”需经项目经理与测试经理共同确认）

### 6.3 跟踪工具

使用 **Jira** 或 **禅道** 进行缺陷全流程追踪，并与自动化测试结果关联。
